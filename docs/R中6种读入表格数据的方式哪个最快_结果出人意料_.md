R中6种读入表格数据的方式哪个最快？结果出人意料！ by 生信宝典
------
<div><h1>R怎么读入表格数据最快？</h1><p>R中有6个常用数据读取函数：</p><ol><li><p><code>utils::read.csv</code>: 默认使用的读入方式 (read.table)</p></li><li><p><code>readr::read_csv</code>: <code>readr</code>包中的读入函数 (RStudio中默认也包含了这一方式)</p></li><li><p><code>data.table::fread</code>: 来自<code>data.table</code>包</p></li><li><p><code>base::load</code>: 加载rda文件</p></li><li><p><code>base::readRDS</code>: 读取二进制数据</p></li><li><p><code>feather::read_feather</code>: 一种新的<code>feather</code>格式的二进制数据</p></li></ol><h2>生成测试数据</h2><pre><code>set.seed(123)<br>df &lt;- data.frame(replicate(10, sample(0:2000, 15 * 10^5, rep = TRUE)),<br>                 replicate(10, stringi::stri_rand_strings(1000, 5)))<br>dim(df)<br><br>[1] 1500000      20</code></pre><p>把数据存储为<code>csv</code>格式，<code>feather</code>格式, <code>RDS</code>格式和<code>Rdata</code>格式。</p><pre><code>path_csv &lt;- 'df.csv'<br>path_feather &lt;- 'df.feather'<br>path_rdata &lt;- 'df.RData'<br>path_rds &lt;- 'df.rds'<br>library(feather)<br>library(data.table)<br>write.csv(df, file = path_csv, row.names = F)<br>write_feather(df, path_feather)<br>save(df, file = path_rdata)<br>saveRDS(df, path_rds)</code></pre><p>计算下各个文件的大小；</p><ul><li><p><code>RDS</code>和<code>RData</code>占的空间最小，不到<code>30M</code></p></li><li><p><code>feather</code>文件占的空间最大，<code>185M</code></p></li><li><p><code>CSV</code>文件占了<code>179M</code>，与<code>feather</code>相差不大</p></li></ul><pre><code>files &lt;- c('df.csv', 'df.feather', 'df.RData', 'df.rds')<br>info &lt;- file.info(files)<br>info$size_mb &lt;- info$size/(1024 * 1024)<br>print(subset(info, select=c("size_mb")))<br><br>##              size_mb<br>## df.csv     179.46119<br>## df.feather 185.96806<br>## df.RData    29.93043<br>## df.rds      29.93044</code></pre><h2>评估速度</h2><p>采用<code>microbenchmark</code>包比较使用不同函数读取所需时间。</p><h3>基于一台破笔记本的速度评估</h3><pre><code>library(microbenchmark)<br>benchmark &lt;- microbenchmark(readCSV = utils::read.csv(path_csv),<br>               readrCSV = readr::read_csv(path_csv, progress = F),<br>               fread = data.table::fread(path_csv, showProgress = F),<br>               loadRdata = base::load(path_rdata),<br>               readRds = base::readRDS(path_rds),<br>               readFeather = feather::read_feather(path_feather), times = 10)<br># save(benchmark, file = "benchmark.rda")</code></pre><ol><li><p>速度最快的是<code>feather::read_feather</code>,但需要预先把数据存储为<code>feather</code>格式。</p><p>其次是<code>fread</code>，但差别不明显。</p></li><li><p><code>load</code>和<code>readRDS</code>没有表现出速度优势，但却需要对文件进行格式转。</p></li><li><p><code>fread</code>函数读取<code>csv</code>的速度最快；</p><p><code>readr::read_csv</code>函数次之；</p><p>默认使用的<code>read.csv</code>速度最慢。</p></li></ol><pre><code># load("benchmark.rda")<br># min最小耗时<br># lq, uq 耗时的第一和第三四分位数<br># max最大耗时<br>summary(benchmark)<br><br>##          expr        min         lq       mean     median         uq       max neval<br>## 1     readCSV 18167.9870 18788.3618 19925.4103 19007.2041 22030.0041 23347.212    10<br>## 2    readrCSV  3464.2280  4000.4398  4275.1156  4244.4979  4323.3464  5383.394    10<br>## 3       fread   942.6006   977.3619  1252.6054  1265.5249  1429.5995  1741.290    10<br>## 4   loadRdata  4759.2474  4797.0618  5409.0172  5128.4355  6019.2456  7179.577    10<br>## 5     readRds  4808.1431  4891.1552  5653.4468  5708.4353  6233.9128  6473.565    10<br>## 6 readFeather   673.4313   739.8717   894.3114   871.7041   934.1121  1351.928    10</code></pre><h3>基于一台SSD硬盘的工作站速度评估</h3><pre><code>library(microbenchmark)<br>benchmark2 &lt;- microbenchmark(readCSV = utils::read.csv(path_csv),<br>               readrCSV = readr::read_csv(path_csv, progress = F),<br>               fread = data.table::fread(path_csv, showProgress = F),<br>               loadRdata = base::load(path_rdata),<br>               readRds = base::readRDS(path_rds),<br>               readFeather = feather::read_feather(path_feather), times = 10)<br>save(benchmark2, file = "benchmark2.rda")</code></pre><ol><li><p>速度最快的是<code>readr::read_csv</code>,其次是<code>feather::read_feather</code>，然后是<code>fread</code>。</p></li><li><p>默认使用的<code>read.csv</code>速度最慢。</p></li></ol><pre><code>load("benchmark2.rda")<br>print(benchmark2, signif = 2)<br><br>## Unit: milliseconds<br>##         expr  min   lq mean median   uq  max neval<br>##      readCSV 9200 9200 9300   9300 9400 9600    10<br>##     readrCSV  240  250  290    260  270  370  410  410    410  440  450    10</code></pre><h3>原始文章的测试结果中，</h3><ol><li><p>速度最快的是<code>feather</code>,但需要预先把数据存储为<code>feather</code>格式。</p><p>数据库保存为<code>feather</code>格式是很快的，适用于需要读取某个大文件或程序运行中计算出的结果时。</p></li><li><p><code>load</code>和<code>readRDS</code>速度稍次，但也需要对文件进行格式转换，优点是存储的文件较小。</p></li><li><p><code>fread</code>函数读取<code>csv</code>的速度最快；</p><p><code>readr::read_csv</code>函数次之；</p><p>默认使用的<code>read.csv</code>速度最慢。</p></li></ol><pre><code>##Unit: 10<br>## readFeather   1.5   1.8   2.988021    3.4   3.6   4.1    10</code></pre><h2>测试结论</h2><ol><li><p>最常用的<code>read.table</code>在每个测试中都是表现最差的。</p></li><li><p>`readr::read_csv` （处理不同编码更合适，<a target="_blank" href="http://mp.weixin.qq.com/s?__biz=MzI5MTcwNjA4NQ==&amp;mid=2247524901&amp;idx=2&amp;sn=cd2247d4d422072217cf1ac9c3f05232&amp;chksm=ec0ea7afdb792eb98964a2fbb149799b99b008505f67e9743a9ccb8c622700e56bd7a10d2f96&amp;scene=21#wechat_redirect" data-itemshowtype="0" tab="innerlink" data-linktype="2">R中读取包含中文字符的文件时这个诡异的错误你见过吗?</a>）和`data.table::fread`可以作为日常使用或读取大表格的首选。<br></p></li><li><p>不同电脑测试结果差别大（可能是软件版本的问题，也可能是硬件特征问题）。别人用着快的你不一定用着快，多尝试。</p></li><li><p>读友评论：readr新版本升级之后使用vroom的内核来进行读取，所以速度非常快。值得注意的是，vroom读取的其实并不是完整的数据，只是一种index，所以在之后的操作(比如rbind合并数据框)会展现出比较慢的情况。适合读入大文件，过滤一部分后，进行操作。</p></li><li><p>读友评论：使用data.table::fread进行读取默认使用多线程操作，在服务器cpu高负荷时会影响读取速度。总的来说，fread和vroom(read_csv/tsv)是我最喜欢的两种方式，跟作者意见一致。</p></li></ol><h2>参考</h2><ol><li><p>https://appsilon.com/fast-data-loading-from-files-to-r/</p></li></ol><p><br></p><h3>往期精品<span>(<span>点击图片直达文字对应教程</span>)</span></h3><table><tbody><tr><td
