data-tool="mdnice编辑器" data-website="https://www.mdnice.com"><p data-tool="mdnice编辑器">SCP的完整安装分为两个部分，一个是R包安装，另一个是内部的python环境构建。</p><p data-tool="mdnice编辑器">如果不需要运行一些python相关的方法则可以跳过下面的[创建SCP的python环境】步骤。</p><hr data-tool="mdnice编辑器"><h2 data-tool="mdnice编辑器"><span></span><span>1、安装到全局环境中</span></h2><p data-tool="mdnice编辑器">这里的全局环境是指R的默认包路径（在R中通过<code>.Library</code>查看），与之相反的是后文中使用renv所创建的隔离环境。</p><p data-tool="mdnice编辑器">R版本要求：</p><ul data-tool="mdnice编辑器"><li><section>R &gt;= 4.1.0</section></li></ul><p data-tool="mdnice编辑器">Python版本要求：</p><ul data-tool="mdnice编辑器"><li><section>Python 3.7-3.9</section></li></ul><p data-tool="mdnice编辑器">注意，实际上Python版本并不重要，因为无需手动配置python环境，使用SCP中的<code>PrepareEnv</code>函数会自动下载安装所需版本的conda、python以及软件包。</p><h3 data-tool="mdnice编辑器"><span></span><span>1.1 R包安装</span><span></span></h3><ol data-tool="mdnice编辑器"><li><section>从GitHub上自动下载安装最新版本的SCP（<span>推荐</span>）:</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code><span>if</span> (!<span>require</span>(<span>"devtools"</span>, quietly = <span>TRUE</span>)) {<br>  install.packages(<span>"devtools"</span>)<br>}<br>devtools::install_github(<span>"zhanghao-njmu/SCP"</span>)<br></code></pre><ol start="2" data-tool="mdnice编辑器"><li><section>手动安装:</section></li></ol><p data-tool="mdnice编辑器">用各种方式下载好R包，比如在R中可以运行命令：</p><pre data-tool="mdnice编辑器"><span></span><code>download.file(<span>"https://github.com/zhanghao-njmu/SCP/archive/refs/tags/v0.5.1.tar.gz"</span>, destfile = <span>"SCP.tar.gz"</span>)<br></code></pre><ul data-tool="mdnice编辑器"><li><section>使用<code>devtools::install_local</code>自动安装SCP及相关依赖（这种安装过程可以自动寻找安装依赖包）：</section></li></ul><pre data-tool="mdnice编辑器"><span></span><code>devtools::install_local(<span>"SCP-0.5.1.tar.gz"</span>)<br></code></pre><ul data-tool="mdnice编辑器"><li><section>或使用<code>install.packages</code>安装SCP（如果缺失依赖包会报错，需要再根据提示手动逐一安装依赖包）：</section></li></ul><pre data-tool="mdnice编辑器"><span></span><code>install.packages(<span>"SCP-0.5.1.tar.gz"</span>, repos = <span>NULL</span>, type = <span>"source"</span>)<br></code></pre><h3 data-tool="mdnice编辑器"><span></span><span>1.2 创建SCP的python环境</span><span></span></h3><p data-tool="mdnice编辑器">当SCP首次安装好之后，运行<code>library(SCP)</code>会有下面的提示：</p><pre data-tool="mdnice编辑器"><span></span><code><span>library</span>(SCP)<br><br><span>## &gt; SCP python environment not found.</span><br><span>## &gt; If you have already created an SCP python environment using conda, you can specify the conda path by setting options(reticulate.conda_binary = "/path/to/conda", SCP_env_name = "SCP_env") before loading the package.</span><br></code></pre><p data-tool="mdnice编辑器">这不是报错，你可以继续使用SCP的大部分功能，也可以运行<code>SCP::PrepareEnv()</code>来继续构建SCP所需的python环境和软件包，来运行<code>RunPAGA</code>、<code>RunSCVELO</code>等python方法。</p><p data-tool="mdnice编辑器">国内用户建议使用镜像，可用的一些镜像地址可在GitHub上找到，一般可以直接运行下面的代码完成创建：</p><pre data-tool="mdnice编辑器"><span></span><code>SCP::PrepareEnv(<br>  miniconda_repo = <span>"https://mirrors.bfsu.edu.cn/anaconda/miniconda"</span>,<br>  pip_options = <span>"-i https://pypi.tuna.tsinghua.edu.cn/simple"</span><br>)<br></code></pre><p data-tool="mdnice编辑器">当想重装python环境时，<code>PrepareEnv</code>还可以通过设置下面的两个参数之一来进行安装：</p><ul data-tool="mdnice编辑器"><li><section><code>force = TRUE</code>表示强制安装，也就是会删除已存在的python环境和软件包</section></li><li><section><code>conda = NULL</code>会重新下载安装miniconda，此时也会删除已存在的python环境并重装</section></li></ul><p data-tool="mdnice编辑器"><code>PrepareEnv</code>函数的过程大致是：</p><ol data-tool="mdnice编辑器"><li><section>寻找环境中可用的conda, 用户也可以手动设置conda路径：</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code>options(reticulate.conda_binary = <span>"/path/to/conda"</span>)<br></code></pre><p data-tool="mdnice编辑器">如果找不到会根据<code>miniconda_repo</code>自动下载miniconda，安装到miniconda到以下路径：</p><ul data-tool="mdnice编辑器"><li><section><p>linux: <code>~/.local/share/r-miniconda</code></p></section></li><li><section><p>osx: <code>~/Library/r-miniconda</code></p></section></li><li><section><p>win: <code>C:/Users/xxx/AppData/Local/r-miniconda</code></p></section></li></ul><ol start="2" data-tool="mdnice编辑器"><li><section>寻找SCP的python环境，默认环境名称是<code>SCP_env</code>，用户也可以手动设置环境名称：</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code>options(SCP_env_name = <span>"new_name"</span>)<br></code></pre><p data-tool="mdnice编辑器">如果找不到环境则自动创建，创建环境时使用的python版本默认为3.8（conda自动下载）</p><ol start="3" data-tool="mdnice编辑器"><li><section>检查环境中的python包版本，这里要求比较严格，以保证各软件兼容，如果找不到对应版本的包，则会使用pip自动下载。这里使用pip安装包而不是用conda的原因有两个，一个是快，另外一个是有些包使用conda自动安装后可能无法使用，对个人环境和权限有要求。</section></li></ol><hr data-tool="mdnice编辑器"><h2 data-tool="mdnice编辑器"><span></span><span>2、安装到隔离的R环境中</span></h2><p data-tool="mdnice编辑器">SCP会牵扯许多依赖包，如果不想干扰当前的环境，比如一些依赖包的版本不想在安装时被改变，或者想将SCP及其依赖包设定在一个固定版本避免更新，来保证重复性，我们可以使用renv。类似于conda环境，R也可以使用renv创建一个独立的R包环境。</p><ol data-tool="mdnice编辑器"><li><section>首先我们要创建一个隔离的R环境：</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code><span>if</span> (!<span>require</span>(<span>"renv"</span>, quietly = <span>TRUE</span>)) {<br>  install.packages(<span>"renv"</span>)<br>}<br>dir.create(<span>"~/SCP_env"</span>, recursive = <span>TRUE</span>) <span>## It cannot be the home directory "~" !</span><br>renv::init(project = <span>"~/SCP_env"</span>, bare = <span>TRUE</span>, restart = <span>TRUE</span>)<br></code></pre><ol start="2" data-tool="mdnice编辑器"><li><section>在隔离环境中安装SCP：</section></li></ol><ul data-tool="mdnice编辑器"><li><section>如果全局环境中没有SCP，则运行：</section></li></ul><pre data-tool="mdnice编辑器"><span></span><code>renv::activate(project = <span>"~/SCP_env"</span>)<br>renv::install(<span>"BiocManager"</span>)<br>renv::install(<span>"zhanghao-njmu/SCP"</span>, repos = BiocManager::repositories())<br>SCP::PrepareEnv(<br>  miniconda_repo = <span>"https://mirrors.bfsu.edu.cn/anaconda/miniconda"</span>,<br>  pip_options = <span>"-i https://pypi.tuna.tsinghua.edu.cn/simple"</span><br>)<br></code></pre><ul data-tool="mdnice编辑器"><li><section>如果全局环境中已经安装好了SCP，则运行：</section></li></ul><pre data-tool="mdnice编辑器"><span></span><code>renv::activate(project = <span>"~/SCP_env"</span>)<br>renv::hydrate(<span>"SCP"</span>)<br>SCP::PrepareEnv(<br>  miniconda_repo = <span>"https://mirrors.bfsu.edu.cn/anaconda/miniconda"</span>,<br>  pip_options = <span>"-i https://pypi.tuna.tsinghua.edu.cn/simple"</span><br>)<br></code></pre><ol start="3" data-tool="mdnice编辑器"><li><section>使用SCP前激活环境:</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code>renv::activate(project = <span>"~/SCP_env"</span>)<br><br><span>library</span>(SCP)<br>data(<span>"pancreas_sub"</span>)<br>pancreas_sub &lt;- RunPAGA(srt = pancreas_sub, group_by = <span>"SubCellType"</span>, linear_reduction = <span>"PCA"</span>, nonlinear_reduction = <span>"UMAP"</span>)<br>CellDimPlot(pancreas_sub, group.by = <span>"SubCellType"</span>, reduction = <span>"draw_graph_fr"</span>)<br></code></pre><ol start="4" data-tool="mdnice编辑器"><li><section>保存SCP环境的状态，当一些包版本改动后可以恢复至保存的状态:</section></li></ol><pre data-tool="mdnice编辑器"><span></span><code>renv::snapshot(project = <span>"~/SCP_env"</span>)<br><span>## The versions of some packages have been modified.</span><br>renv::restore(project = <span>"~/SCP_env"</span>)<br></code></pre><p data-tool="mdnice编辑器">关于隔离环境的详细使用说明，请参考renv的文档：https://rstudio.github.io/renv/articles/renv.html</p><hr data-tool="mdnice编辑器"><h2 data-tool="mdnice编辑器"><span></span><span>3、常见问题</span></h2><h3 data-tool="mdnice编辑器"><span></span><span>3.1、ERROR: dependency ‘xxx’ is not available for package ‘SCP’</span><span></span></h3><p data-tool="mdnice编辑器">这是因为安装依赖于一些Bioconductor的包，但是找不到可以该包所在的仓库。</p><p data-tool="mdnice编辑器">解决办法是在R中运行：</p><pre data-tool="mdnice编辑器"><span></span><code><span>if</span> (!<span>require</span>(<span>"BiocManager"</span>, quietly = <span>TRUE</span>)) {<br>  install.packages(<span>"BiocManager"</span>, lib = lib)<br>}<br>options(repos = BiocManager::repositories())<br></code></pre><p data-tool="mdnice编辑器">之后再安装SCP或依赖包。</p><h3 data-tool="mdnice编辑器"><span></span><span>3.2、Timeout was reached: [api.github.com] Connection timed out after 10005 milliseconds</span><span></span></h3><p data-tool="mdnice编辑器">下载连接超时，在R中设置”options(timeout=10000)“或者手动下载安装包进行安装，参考上文[R包安装]</p><h3 data-tool="mdnice编辑器"><span></span><span>3.3、namespace ‘xxx’ xxx is already loaded, but &gt;= xxx is required.</span><span></span></h3><p data-tool="mdnice编辑器">简单来说，就是”当前的R session中已经载入了一个旧的包，需要将其升级”。</p><p data-tool="mdnice编辑器">一般情况下按照提示<code>install.packages('xxx')</code>就可以了，但是很多人可能发现问题依然会出现。</p><p data-tool="mdnice编辑器">也会有很多人发现一些蹊跷的地方，比如：</p><ul data-tool="mdnice编辑器"><li><section><p>自己根本没有加载过这个当事”包”</p></section></li><li><section><p>当事”包”版本明明已经符合，但是仍然报错</p></section></li></ul><p data-tool="mdnice编辑器">实际上，这个问题常出现在Rstudio中。在Rstudio打开一个新的session，按几个回车没那么快反应出命令提示符<code>&gt;</code>的话，那应该是你的rstudio正在帮你载入一些包，其中就会包括这些当事”包”。</p><p data-tool="mdnice编辑器">所以一步到位的解决办法就是：打开R console（而非Rstudio）–&gt;install.packages(‘xxx’) –&gt; 再次安装SCP或者你所需要的包</p><p data-tool="mdnice编辑器">如果想在Rstudio中避免启动session后就载入一些包，可以取消Rstudio所有的启动后加载功能：打开Tools–&gt; Global Options –&gt; General，将下图的选项框全部取消勾选：</p><figure data-tool="mdnice编辑器"><img data-ratio="0.5425925925925926" data-src="https://mmbiz.qpic.cn/mmbiz_png/siaia0BDGJdjR6odEia7csT9oiceu4Pqf7RfyhGowrgQhQQB0iax3icmtTmBxH0yQdv5V09CDqiacdPJIict6tbD87p4PA/640?wx_fmt=png" data-type="png" data-w="1080" data-tool="mdnice编辑器"><span></span><span>3.4、Error: package or namespace load failed for ‘dbplyr’</span><span></span></h3><p data-tool="mdnice编辑器">问题和之前的<code>namespace 'xxx' xxx is already loaded, but &gt;= xxx is required.</code>是类似的，Rsession启动时被预先加载了一些包，导致无法正常加载<code>dbplyr</code>。</p><p data-tool="mdnice编辑器">有两种解决办法：</p><ol data-tool="mdnice编辑器"><li><section><p>在R console加载运行SCP，或者取消Rstudio所有的启动后加载功能；</p></section></li><li><section><p>在你的home目录下或者R project目录下,新建<code>.Rprofile</code>（它是R的启动文件，会在R环境启动时自动加载），将<code>library(dbplyr)</code>或者直接将<code>library(SCP)</code>写入其中即可。</p></section></li></ol><h3 data-tool="mdnice编辑器"><span></span><span>3.5、‘…SCP_env/lib/libpython3.8.dylib’ (mach-o file, but is an incompatible architecture (have ‘arm64’, need ‘x86_64’))</span><span></span></h3><p data-tool="mdnice编辑器">conda的版本与处理器架构不符，需要x86_64而当前安装的是arm64，解决办法是重新下载安装miniconda/anaconda，或者在R中使用以下命令重装miniconda:</p><pre data-tool="mdnice编辑器"><span></span><code>SCP::PrepareEnv(<br>  conda = <span>NULL</span>,<br>  miniconda_repo = <span>"https://mirrors.bfsu.edu.cn/anaconda/miniconda"</span>,<br>  pip_options = <span>"-i https://pypi.tuna.tsinghua.edu.cn/simple"</span>,<br>  force = <span>TRUE</span><br>)<br></code></pre><h3 data-tool="mdnice编辑器"><span></span><span>3.6、已经顺利安装好了SCP的python环境，但是library(SCP)后仍然提示”SCP python environment not found.”</span><span></span></h3><p data-tool="mdnice编辑器">这是因为一个R session下通过reticulate对python的调用是单次的。</p><p data-tool="mdnice编辑器">如果此前已经运行<code>library(reticulate)</code>调用了一个python，且该python不是SCP所用的python（版本不符合或当中没有SCP的python环境），则需要重新启动一个新的Rsession再运行<code>library(SCP)</code>。</p></section><section><p><br></p><section
