文件。则在安装该包时，这行代码将被执行一次，此时 <code>time</code> 变量将储存下当前时间（即安装的时间）。随后， <code>time</code> 变量将作为一个对象，和它的值一起被保留下来。在你 library 该包时， <code>time</code> 代表的对象会和它的值被一起导入。此时查看 <code>time</code> 包，它能够被正常的编译安装，并实现所需要的功能。但还需要解决的问题是，如何让你的包可以在别人的电脑环境下运行，以及如何让别人知道怎样使用你的函数。</p><blockquote data-tool="mdnice编辑器"><p>关于运行环境：</p><p>在包的功能比较单一，涉及的函数比较少的时候，环境问题主要就是包的依赖问题。比如说，素材中的 <code>plot.chain</code> 函数，需要调用 <code>ggplot2</code> 包中的某些函数。那么，如果别人电脑上还没有安装 <code>ggplot2</code> 包，我们的函数在 <code>library</code> 时，就会报错。 R 包开发的很重要的一个问题就是解决这样的包依赖问题。解决方法在 2.1 节 (5) 中。</p></blockquote><h1 data-tool="mdnice编辑器"><span></span>2 修改说明文档</h1><p data-tool="mdnice编辑器">第一小节所构建的 R 包项目已经可以实现我们所需要的功能，这一小节主要是解决包的各种描述性信息以及环境依赖问题。这是为了保证新用户能够快速理解 R 包的使用方法，以及保证 R 包能够正常工作。</p><h2 data-tool="mdnice编辑器"><span></span>2.1 DESCRIPTION</h2><blockquote data-tool="mdnice编辑器"><p>关于文件格式：</p><p>事实上，整个项目里面的所有文件都是文本形式的文件。例如 代码的 R 文件， 说明文档的 Rd 文件，DESCRIPTION 和 NAMESPACE 文件。</p></blockquote><p data-tool="mdnice编辑器">DESCRIPTION 是一个文本文件，每一行对应于一类特定信息。我们可以使用 Rstudio 打开，也可以使用 txt、VScode 等文本编辑器打开。打开后我们逐行来看其对应的信息。</p><pre data-tool="mdnice编辑器"><span></span><code>Package: randomwalkli<br>Type: Package<br>Title: randomwalk<br>Version: 0.1.0<br>Authors@R: person(<span>'Hadley'</span>,<span>'Wickham'</span>,email = <span>'...'</span>,role = c(<span>'aut'</span>,<span>'cre'</span>))<br>Maintainer: The package maintainer &lt;yourself@somewhere.net&gt;<br>Description: to plot ....<br>License: CC0<br>Encoding: UTF-8<br>LazyData: <span>true</span><br>Imports: <br>    ggplot2<br>RoxygenNote: 7.2.2<br><br></code></pre><p data-tool="mdnice编辑器">第一行是包的名称，类似于 "ggplot2" "MASS"。包名将完全取决于此，但不建议随便修改，建议和文件夹名称、项目名称，以及未来GitHub上的仓库名称对应。</p><p data-tool="mdnice编辑器">第二行是项目类型，"Package" 是指这是一个包。不可随意修改。</p><h3 data-tool="mdnice编辑器"><span></span>(1) 标题和描述<span></span></h3><p data-tool="mdnice编辑器">第三行和第七行分别是 Title 和 Description，是对包的功能的一个描述。其中标题相对简短，描述更加详细。如下是包 ggplot2 的标题和描述，可以看到是对包功能的介绍。</p><pre data-tool="mdnice编辑器"><span></span><code>Title: Create Elegant Data Visualisations Using the Grammar of Graphics<br>Description: A system <span>for</span> <span>'declaratively'</span> creating graphics,<br>    based on <span>"The Grammar of Graphics"</span>. You provide the data, tell <span>'ggplot2'</span><br>    how to map variables to aesthetics, what graphical primitives to use,<br>    and it takes care of the details.<br></code></pre><p data-tool="mdnice编辑器">这一部分手动修改并保存即可。</p><h3 data-tool="mdnice编辑器"><span></span>(2) Version<span></span></h3><p data-tool="mdnice编辑器">第四行 Version（版本），由三部分构成： 大版本，子版本，补丁版本，由点号隔开。</p><p data-tool="mdnice编辑器">如果要上传到 CRAN 或其他的大型平台，版本号要求比较严格。例如 <code>ggplot2</code> 包的最新版本已经是 3.3.6 。而我们一般上传到 GitHub 上，版本就没有人进行规范，所以简单注意即可。</p><h3 data-tool="mdnice编辑器"><span></span>(3) Author and Maintainer<span></span></h3><p data-tool="mdnice编辑器">第五行作者，显而易见就是给出包的作者信息。作者信息有着非常标准的书写格式，即 Authors@R: c(person(), person(),...)。</p><p data-tool="mdnice编辑器">事实上，这是一段 R 中的可执行代码（@R: 应该就是指在R中可以被 call 并执行），所以一位作者的信息可以由 R 的一个函数给定，其中包含的参数如下所示。因此如果是多位作者，就需要使用 <code>c()</code> 来创建一个向量。示例如下：</p><pre 显示，该信息在后续会由 Author 中的信息自动生成。因此，保证 author 里面有且只有一个人的角色包含 cre ，且有邮箱即可。</p><p data-tool="mdnice编辑器">Authors 部分的信息可以手动修改。</p><h3 data-tool="mdnice编辑器"><span></span>(4) License<span></span></h3><p data-tool="mdnice编辑器">第八行是包的版权许可声明，表明这个包的版权许可类型，即他人使用该包的信息时，必须遵守的版权规定。包括但不限于：</p><ul data-tool="mdnice编辑器"><li><section>MIT 其允许人们使用和发布包的代码，唯一的限制是许可证必须和代码一起发布</section></li><li><section>GPL-2 意味着任何包含你的代码的包都必须使用GPL兼容的许可证来发布，此外，任何人发布代码的修改版本都必须公布源代码</section></li><li><section>CC0 这个许可证放弃了你对代码和数据的所有权，任何人都可以自由的把它用于任何地方</section></li></ul><p data-tool="mdnice编辑器">可以手动敲上去，但建议使用代码 <code>usethis::use_mit_license()</code> 或者  <code>usethis::use_cc0_license()</code> 等。</p><h3 data-tool="mdnice编辑器"><span></span>(5) 依赖包<span></span></h3><p data-tool="mdnice编辑器">第十一行是在声明我们的 R 包所需要的依赖包。这是在解决 1.4 节 提到的环境问题。</p><p data-tool="mdnice编辑器">该问题的起点是，你不能保证我们所需要的某些包，例如我们使用了的 ggplot2 包，在用户的电脑上已经安装过了。如果他没有安装 ggplot2 ，但你却在你的函数里面调用了 ggplot2 的函数，那么显然会报错。</p><p data-tool="mdnice编辑器">所以我们需要在此处声明依赖的包。声明之后，当其他人安装这个 R 包时，会同时检查是否安装有依赖的包，如果没有则默认帮你自动安装。</p><p data-tool="mdnice编辑器">对该部分进行修改，我们建议使用函数<code>usethis::use_package("packagename")</code> ，例如 <code>usethis::use_package("ggplot2")</code> ，当然也可以按照格式，手动输入（不建议）。</p><blockquote data-tool="mdnice编辑器"><p>关于<code>usethis::use_package("packagename")</code> 的参数：</p><p>建议使用 <code>?usethis::use_package</code> 查看帮助文档，因为该函数提供了非常丰富的选项，以实现对依赖包的管理，比如说控制依赖包的版本。</p></blockquote><blockquote data-tool="mdnice编辑器"><p>关于依赖包和导入包的区别：</p><p>这里添加的依赖包，只意味着用户在安装这个R包时，会检查是否安装了依赖包，如果没有，则帮助安装。但是在函数内部调用的时候，仍然需要重新导入 <code>library(ggplot2)</code> ，或者在NAMESPACE 中声明。</p></blockquote><h2 data-tool="mdnice编辑器"><span></span>2.2 文档</h2><p data-tool="mdnice编辑器">一个好的文档应当帮助使用者更快地上手你的包的功能 / 函数。</p><h3 data-tool="mdnice编辑器"><span></span>2.2.1 函数文档<span></span></h3><p data-tool="mdnice编辑器">几乎所有的 R 包的函数都有相应的函数文档，它能够帮助使用者了解函数实现的功能、传入参数、返回值，并提供案例、参考文献等诸多信息，以帮助使用者快速地了解该函数的用法。</p><p data-tool="mdnice编辑器">以最简单的均值函数 <code>mean</code> 为例，使用 <code>help(mean)</code> 或者<code>?(mean)</code> 代码，就可以找到它的帮助文档。其中给出了 描述(description)、 用法(usage)、 参数(arguments)、 返回值(value)、 参考文献(references)、 相似内容(see also)、 案例(examples)。</p><figure data-tool="mdnice编辑器"><img data-tool="mdnice编辑器">在需要帮助文档的函数上方书写注释，注释与帮助文档的条目一一对应。此外，为了与 R 文件以 # 开头的注释相区分，函数的注释使用 #‘ 开头。例如：</p><pre data-tool="mdnice编辑器"><span></span><code><span>#' Plot random walk chains</span><br><span>#'</span><br><span>#' It is an example of create a R package. This function is used to</span><br><span>#' make a set of random walk chains. We hope to</span><br><span>#' make it easy to plot random walk chains.</span><br><span>#' Thank you for using it.</span><br><span>#'</span><br><span>#' @param p.length length of the generated chain.</span><br><span>#' @param p.prob probability to get positive in once walk.</span><br><span>#' @param p.num number of chains to generate.</span><br><span>#' @param p.seed random seed.</span><br><span>#'</span><br><span>#' @return  No return but plot the figure.</span><br><span>#' @examples</span><br><span>#' main(100,0.5,5,1)</span><br><span>#' main(100,0.51,5,1)</span><br><span>#' main(100,0.4,5,1)</span><br><span>#' main(100,0.6,5,1)</span><br><span>#' @export</span><br><span>#' @importFrom ggplot2 ggplot aes geom_line geom_abline</span><br>main = <span>function</span>(p.length,p.prob,p.num,p.seed = <span>1</span>){<br>  dt.plot = gene.chain(p.length,p.prob,p.num,p.seed)<br>  toplot = plot.chain(dt.plot)<br>  plot(toplot)<br>}<br></code></pre><ol data-tool="mdnice编辑器"><li><section>注释的前两个非空部分分别对应于函数的标题和描述：</section></li></ol><ul><li><section>第一行是函数的标题："Plot random walk chains"</section></li><li><section>与标题间隔一个空行的是函数的描述："It is an example ... ... for using it."</section></li></ul><li><section>除了标题和描述外，其余项目使用 @<itemname>来标识后面跟随的内容。比如说：</itemname></section></li><ul><li><section>参数：格式为 <code>#' @param &lt;para.name&gt; description of the parameter</code> 即 @param 后空一格是参数的名称，再空一格就可以书写该参数的描述。该部分对应于帮助文档中的 Arguments 的部分。</section></li><li><section>返回值：格式为 <code>@return Description of the return value.</code> 即 @return 是指该函数依赖于其余包或者其余包中的某些函数，那么下一次在 library 我们的包时，这些包或者函数也会被导入。</section></li><blockquote data-tool="mdnice编辑器"><p>关于导入和导出模块：</p><p>该模块并非制作一个 R 包必须掌握的内容。我们会在2.3节，有关NAMESPACE 的部分讲解如何绕过这两部分。</p></blockquote><h4 data-tool="mdnice编辑器"><span></span>2.2.1.2 创建文档<span></span></h4><p data-tool="mdnice编辑器">在书写完某个函数的注释后，我们就可以使用命令 <code>devtools::document()</code> 来创建帮助文档。该函数将遍历所有的 R 文件，并自动创建（或更新）所有的帮助文档（以及 NAMESPACE 文件）。</p><p data-tool="mdnice编辑器">所有的帮助文档都会被保存在 <code>./man</code> 路径下，文件名是函数名 <code>function.Rd</code> 。这是一个 Rd 格式的文本文件，使用 Rstudio 或 任何文本编辑器打开，然后会发现这是一个类似于markdown的文件格式。</p><figure data-tool="mdnice编辑器"><img data-ratio="0.6234096692111959" data-src="https://mmbiz.qpic.cn/mmbiz_png/SlSsfDYwjKib8WJGzsBibicszRib6QUwEpRLVucIRrxibjDD9xaGeVUhbb1Qxvus3Ft5diauMMMTbYiaJsITyqKYxPtWQ/640?wx_fmt=png" data-type="png" data-w="786" 也不会检查文档是怎么产生的。所以另外一种创建的文档的方式自然是，自己按照相同的格式书写注释，或者在已有的文档上进行修改。当然，我们并不建议如此操作，因为下一次运行  <code>devtools::document()</code> 函数时，会更新所有的帮助文档。</p><blockquote data-tool="mdnice编辑器"><p>事实上，如果从 Rstudio 中打开，它会提示你该文档由函数 <code>devtools::document()</code> ，基于另一文件生成，因此在 Rstudio 中是只读 (read only) 的，以避免被误改。</p></blockquote><p data-tool="mdnice编辑器">通过不同的关键词 <code>@&lt;keyword&gt;</code> ，我们还可以添加其它很多类型的内容。例如， <code>?mean</code> 中我们可以看到的 reference （参考文献）, see also （相关指引）等。可以根据自己的需求，查看说明文档或教程。</p><h3 data-tool="mdnice编辑器"><span></span>2.2.2 包文档等非函数文档<span></span></h3><p data-tool="mdnice编辑器">事实上，不仅函数有自己的说明文档，包 (package) 自身也有说明文档。使用命令 <code>help(ggplot2)</code> ，我们就可以看到 ggplot2 的说明文档。那么如何创建包的文档呢？</p><p data-tool="mdnice编辑器">包的文档创建类似于函数文档，也是写在 .R 文件中，然后使用命令 <code>devtools::document</code> 时，会自动生成相应的文档。因此我们只需要了解 包的说明文档的书写格式即可。</p><p data-tool="mdnice编辑器">首先，在 R 文件夹中，创建一个空的 <code>.R</code> 文件，并写入一段同样以 <code>#'</code> 开头的注释。与函数相同，该部分注释内容的每一行同样对应于特定的信息。与函数文档的区别是，注释下方不再是某个函数，而是空值 <code>NULL</code> 。例如：</p><pre data-tool="mdnice编辑器"><span></span><code><span>#' randomwalkli : package to learn how to develop R package</span><br><span>#'</span><br><span>#' It is an example for R package development.</span><br><span>#' The main function of it is to create a plot,</span><br><span>#' which is consist of several random walk chains.</span><br><span>#'</span><br><span>#'</span><br><span>#' @section 1 The purpose of package:</span><br><span>#' Function main is used to ... The key function is function main ..........</span><br><span>#'</span><br><span>#' @section 2 Conclusion:</span><br><span>#' We hope this case can help you learn about the development of a R package.</span><br><span>#'</span><br><span>#' @docType package</span><br><span>#' @name randomwalkli</span><br><br><span>NULL</span><br><br></code></pre><ul data-tool="mdnice编辑器"><li><section><p>第一行是标题 (title)，与标题相隔一个空行的是描述 (description)。</p></section></li><li><section><p>之后的每一部分仍然使用 @ 加关键词指定内容属性：</p></section></li><ul><li><section>例如 <code>@section</code> ，类似 Latex 中的 section。其中关键词后空一格是这个 section 的标题，标题以冒号 <code>:</code> 结尾。换一行书写 section 的内容。</section></li></ul><li><section><p>最后两行分别是文档类型 (docType) 和 名称 (name)：</p></section></li><ul><li><section>文档类型为 package，指这个说明文档给出的是 package 的信息。</section></li><li><section>名称一般就是包的名称，也是搜索这个说明文档时所需要的名称。例如 <code>help("randomwalkli")</code> 。如果 name 位置书写的是 <code>rwl</code> ，则需要使用命令 <code>help("rwl")</code> 搜索该篇说明文档。</section></li></ul></ul><blockquote data-tool="mdnice编辑器"><p>关于 文档的 name 属性：</p><p>如果仔细思考上述关于包的名称和文档类型的说明之后就会发现，并不是某个文档和包绑定了，只是而是恰好有一个和包名字相同的文档。因此，当我们用 <code>help</code> 命令搜索包的名称时，就刚好会找到这个文档。所以我们会在这个文档里面写包的说明文档。</p><p>同样的思路可以为其他对象添加相应的文档，比如说类(class)、数据集等。此外，文档也有更多的关键词以实现较为复杂的文本格式。</p></blockquote><h2 data-tool="mdnice编辑器"><span></span>2.3 NAMESPACE</h2><h3 data-tool="mdnice编辑器"><span></span>2.3.1 绕开 NAMESPACE 文件<span></span></h3><blockquote data-tool="mdnice编辑器"><p>关于NAMESPACE 的导入和导出：</p><p>事实上，在制作一个功能相对单一的 R 包时，我们可以不需要修改该文件。因为无非是</p><p>(1) 不指定导出的函数，而是把所有函数导出。</p><p>(2) 不指定导入的函数/包，而是在函数里面写 <code>library</code> 来导入，或者 <code>package::function()</code> 的形式来使用依赖包的函数。</p></blockquote><p data-tool="mdnice编辑器">导入依赖包的两种简单方法：</p><pre data-tool="mdnice编辑器"><span></span>2.3.2 不绕开 NAMESPACE 文件<span></span></h3><h4 data-tool="mdnice编辑器"><span></span>(1) 内容<span></span></h4><p data-tool="mdnice编辑器">该文件包含两部分内容：</p><ul data-tool="mdnice编辑器"><li><section>你希望从自己的包里面导出去 (export) 的函数</section></li><ul><li><section>某些函数只是过程量，用户用不到。如果这样的函数比较多，而且全部导进去的话，不方便用户使用。</section></li><li><section>这些函数往往我们不会为其书写说明文档。</section></li><li><section>不是所有的函数都需要公开给使用者，某些函数可能只是会被下一个函数调用。</section></li><li><section>为什么只导出部分函数：</section></li><li><section>比如素材中，我们只希望用户使用
